# Phase 0: bk-init - Project Setup & PM Mindset

## Overview
- **Priority:** P0
- **Status:** pending
- **Skill:** `bk-init`
- **Approach:** TDD

## Pain Point
> "Mỗi project khác nhau, cần setup riêng về cách quản lý, focus areas, warning triggers"

## Input
```
/bk-init
```

## Expected Output
- Interactive setup wizard
- Generated `project-context.yaml` với user input
- PM mindset configured based on project type

## Key Features

### 1. Interactive Setup Wizard
```
┌─────────────────────────────────────────────────┐
│           Welcome to BrseKit Setup!             │
├─────────────────────────────────────────────────┤
│ Step 1: Project Info                            │
│   → Project name?                               │
│   → Backlog project key?                        │
│                                                 │
│ Step 2: Project Type                            │
│   ○ Project-based (Waterfall) - 請負            │
│   ○ Labor (Agile) - 派遣/SES                    │
│   ○ Hybrid                                      │
│                                                 │
│ Step 3: Customer Profile                        │
│   → Customer name?                              │
│   → Industry?                                   │
│   → Communication style? (formal/casual)        │
│                                                 │
│ Step 4: Customize Focus Areas (optional)        │
│   → Add custom warning triggers?                │
│   → Modify PM checklist?                        │
└─────────────────────────────────────────────────┘
```

### 2. PM Templates (Base)

| Template | Focus Areas |
|----------|-------------|
| `waterfall.yaml` | CR tracking, Budget, Schedule, Milestones |
| `agile.yaml` | Quality, Priority, Communication, Sprint goals |
| `hybrid.yaml` | Both: CR + Quality, Schedule + Sprint |

### 3. Output: project-context.yaml

```yaml
# Generated by /bk-init on 2026-01-29
project:
  name: "ABC Corp Payment System"
  backlog_key: "ABC-PAY"
  type: "project-based"
  methodology: "waterfall"

customer:
  name: "ABC Corp"
  industry: "Finance"
  timezone: "JST"
  communication_style: "formal"

focus_areas:
  primary:
    - change_request_tracking
    - budget_monitoring
    - milestone_delivery
  secondary:
    - documentation_quality

warning_triggers:
  high:
    - "scope change"
    - "追加要件"
    - "budget"
    - "予算"
    - "delay"
    - "遅延"
  medium:
    - "clarification"
    - "確認"

pm_checklist:
  weekly:
    - "Review CR status"
    - "Check budget burn rate"
  meeting:
    - "Confirm scope alignment"
    - "Track action items"
```

## Key Test Cases
1. Generate valid project-context.yaml từ user input
2. Load base template (waterfall/agile/hybrid) correctly
3. Merge user customizations với base template
4. Validate required fields (project name, backlog key)
5. Handle missing optional fields gracefully

## Dependencies
- Nulab Backlog API (validate project key)
- pm-templates/ (base templates)

## Success Criteria
- [ ] Interactive wizard collects all required info
- [ ] project-context.yaml generated correctly
- [ ] Base templates load and merge properly
- [ ] Other bk-* skills can load context via context_loader.py
